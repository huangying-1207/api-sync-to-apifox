API 同步到 Apifox 工具使用帮助

## 命令

### config init
初始化配置文件，自动从 MCP 凭据加载项目信息

用法：
  api-sync-to-apifox config init [--source-path <路径>] [--framework <框架>]

### scan
扫描后端接口变更（不执行同步）

用法：
  api-sync-to-apifox scan [参数]

参数：
  --source-type <swagger|code>          接口源类型（必填）
  --source-path <路径>                  源路径，代码目录或 Swagger URL（必填）
  --framework <springboot|nodejs|django> 后端框架（source-type 为 code 时必填）
  --scan-type <all|changed>             扫描类型（默认: changed）

### sync
同步后端接口到 Apifox

用法：
  api-sync-to-apifox sync [参数]

参数：
  --project-name <名称>                 MCP 项目名称（已连接时可替代 apifox-project-id 和 apifox-api-key）
  --apifox-project-id <ID>              Apifox 项目 ID
  --apifox-api-key <密钥>               Apifox API 密钥
  --source-type <swagger|code>          接口源类型（必填）
  --source-path <路径>                  源路径（必填）
  --framework <springboot|nodejs|django> 后端框架（source-type 为 code 时必填）
  --sync-mode <incremental|full>        同步模式（默认: incremental）
  --trigger-mode <auto|manual>          触发模式（默认: auto）
  --api-method <方法>                    单独接口同步的 HTTP 方法
  --api-path <路径>                      单独接口同步的路径
  --apis <接口列表>                      多接口同步，格式: "GET:/api/users,POST:/api/users"

### mcp
启动 MCP 交互式控制台

用法：
  api-sync-to-apifox mcp

MCP 命令：
  connect <名称> <项目ID> <API密钥>     连接 Apifox 项目
  disconnect <名称>                      断开连接
  status                                 显示连接状态
  apis <名称>                            获取项目接口列表
  info <名称>                            显示项目详情
  help                                   显示帮助
  exit                                   退出控制台

## 同步模式

  incremental  增量同步，只同步 Git 变更的接口（默认）
  full         全量更新，同步所有接口
