API 同步到 Apifox 工具使用帮助

## 命令

### config init
初始化配置文件，自动从 MCP 凭据加载项目信息

用法：
  api-sync-to-apifox config init [--source-path <路径>] [--framework <框架>]

### scan
扫描后端接口变更（不执行同步）

用法：
  api-sync-to-apifox scan [参数]

参数：
  --source-type <swagger|code>          接口源类型（必填）
  --source-path <路径>                  源路径，代码目录或 Swagger URL（必填）
  --framework <springboot|nodejs|django> 后端框架（source-type 为 code 时必填）
  --scan-type <all|changed>             扫描类型（默认: changed）
  --quiet                               减少日志输出
  --json                                JSON 格式输出

### branches
查询当前 Apifox 项目的迭代分支列表（确认同步前使用）

用法：
  api-sync-to-apifox branches [参数]

参数：
  --project-name <名称>                 MCP 项目名称
  --apifox-project-id <ID>              Apifox 项目 ID
  --apifox-api-key <密钥>               Apifox API 密钥
  --json                                输出 JSON（含分支 ID，供 Agent 写入同步计划）
  --refresh-branches                    忽略缓存，强制重新查询分支

### workflow
一键工作流：scan + branches（不执行 sync）

用法：
  api-sync-to-apifox workflow [参数]

产出：
  - temp/apifox-sync-plan.json（status: pending，作废旧确认）
  - temp/apifox-sync-plan.md
  - 分支列表 JSON 输出到 stdout（供 Agent 读取）

### sync
同步后端接口到 Apifox

用法：
  api-sync-to-apifox sync [参数]

参数：
  --project-name <名称>                 MCP 项目名称（已连接时可替代 apifox-project-id 和 apifox-api-key）
  --apifox-project-id <ID>              Apifox 项目 ID
  --apifox-api-key <密钥>               Apifox API 密钥
  --source-type <swagger|code>          接口源类型（必填）
  --source-path <路径>                  源路径（必填）
  --framework <springboot|nodejs|django> 后端框架（source-type 为 code 时必填）
  --sync-mode <incremental|full>        同步模式（默认: incremental）
  --sync-plan <路径>                    同步计划文件（默认: temp/apifox-sync-plan.json）
  --apifox-branch-id <ID>               指定 Apifox 迭代分支 ID
  --apifox-branch-name <名称>            指定 Apifox 分支名称（推荐）
  --no-branch-prompt                    跳过交互式分支选择，使用默认主分支
  --save-doc                            将 OpenAPI 文档保存到 temp/formatted-api-doc.json（调试用）
  --trigger-mode <auto|manual>          触发模式（默认: auto）
  --api-method <方法>                    单独接口同步的 HTTP 方法
  --api-path <路径>                      单独接口同步的路径
  --apis <接口列表>                      多接口同步，格式: "GET:/api/users,POST:/api/users"
  --quiet                               减少日志输出
  --json                                JSON 格式输出（branches 等）

### mcp
启动 MCP 交互式控制台

用法：
  api-sync-to-apifox mcp

MCP 命令：
  connect <名称> <项目ID> <API密钥>     连接 Apifox 项目
  disconnect <名称>                      断开连接
  status                                 显示连接状态
  scan [项目名] [--参数...]              扫描接口变更（等同 CLI scan）
  sync [项目名] [--参数...]              同步到 Apifox（等同 CLI sync）
  apis <名称>                            获取项目接口列表
  info <名称>                            显示项目详情
  help                                   显示帮助
  exit                                   退出控制台

## 同步模式

  incremental  增量同步，仅同步 apifox-sync-plan.json 中已确认的 syncApis（默认）
  full         全量更新，同步所有接口

## Apifox 分支

  同步前会查询项目分支列表，按**分支名称**让用户选择，默认主分支。
  用户界面通常看不到分支 ID，因此确认时只需回复分支名（如 `master`、`dev`）。

  查询分支：
    node dist/index.js branches --json

  确认同步时，将用户选择的分支名和对应 id 写入 apifox-sync-plan.json：

    "targetBranch": { "id": 1234568, "name": "dev", "isMain": false }
